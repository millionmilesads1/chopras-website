## Implementation Pattern

All internal links sections follow this consistent structure:

```tsx
{/* INTERNAL LINKS SECTION */}
<section className="bg-[#FFFAF5] py-16">
  <div className="max-w-4xl mx-auto px-4 sm:px-6 lg:px-8">
    <h2 className="font-heading text-3xl md:text-4xl text-[#1B2B5E] mb-10">
      {isNl ? '[NL Title]' : '[EN Title]'}
    </h2>
    <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
      {/* 4 Link Cards */}
      <Link href={`${base}/[dish-1]`} className="...">
        {/* Card content */}
      </Link>
      {/* Additional cards... */}
    </div>
    {/* Descriptive paragraph and footer links */}
  </div>
</section>
```

**Standard Elements**:
- Background: `bg-[#FFFAF5]` (warm white) or `bg-white`
- Grid: `grid-cols-1 md:grid-cols-2 gap-6`
- 4 card grid linking to related dishes/services
- Bilingual (isNl) section titles
- Contextual anchor text per page theme
- Bottom footer with home link and contact CTA

## Linking Strategy

### Cross-Page Connections
- **Dish pages** → Related dishes + catering option
- **Event pages** (Diwali, birthday, venue) → Complementary dishes + catering
- **Street food pages** (chaat, pani-puri) → Related chaat dishes + catering
- **Specialty pages** (vegan, vegetarian) → Similar dietary options + catering

### Link Patterns
1. Each page links to 3 other den-haag dishes
2. Every page includes catering link as conversion opportunity
3. Catering link always uses standard CTA copy: "Wilt u dit gerecht voor uw evenement? Indiaas catering Den Haag"
4. All dish links use contextual, page-relevant anchor text (e.g., "event favourite", "vegetarian option")
